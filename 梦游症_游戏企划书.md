## 1. 游戏概述

### 1.1 基本信息

| 项目       | 内容                      |
| -------- | ----------------------- |
| **游戏名称** | 梦游症（Somnambulism）      |
| **类型**   | 2D 实时策略 + 生物收集养成         |
| **核心模式** | PvE 站位塔防出怪（生物固定站位自动攻击，敌人出波次进攻）   |
| **养成内核** | 多样性驱动（收集广度 + 搭配策略）      |
| **目标平台** | 手机（iOS / Android）       |
| **玩家模式** | 单人为主                     |

### 1.2 核心卖点

- **末世 × 卡通反差美学**：末日废土背景配上动漫卡通温暖画风，形成强烈视觉张力；
- **站位塔防出怪**：生物固定站位自动攻击最近目标，敌人沿路线出波次进攻——普通敌人打你的生物，特殊敌人直攻城堡核心；
- **多样性为王**：不升级不进化，策略深度来自"你有多少种生物 + 你怎么搭配"，每只生物的定位独一无二；
- **生死赌桌**：生物 HP 归零即死亡，死亡需要金币+稀有道具复活——每次出战都是一场风险博弈；
- **派系化学反应**：5 阵营 × 10 跨派系反应，混搭比纯阵营更有爆发力。

### 1.3 一句话介绍

> 在末鸣纪的废土上，一个被称为"梦游者"的旅人驾驶移动城堡穿行六大陆，收集各派系生物组建战队。生物不会升级，但死亡会真的来临——你要用收集的多样性打败敌人，用治疗的时机阻止死亡，用搭配的化学反应创造奇迹。

## 2. 世界观设定

> 保留 v0.1 全部设定，无变更。

### 2.1 背景故事

世界进入了被称为 **"末鸣纪"** 的时代。

大灾变之后，文明的骨架已经崩塌，人类没有被外星入侵者消灭，也没有被病毒击倒——他们被自己打垮了。

派系之争是末世的本质：资源、信仰、恐惧、傲慢，在旧文明碎片上孕育出了五支截然不同的势力。他们各自占据一片大陆，彼此敌视，偶尔交战，并在中间那片无名混战大陆里互相渗透与倾轧。

玩家扮演一名末世界原住民——**"梦游者"**。

**梦游者**不属于任何派系，却又游走于所有派系之间。他们的名字来源于一个末世传说：*"在世界崩塌时入睡，在世界重建时行走"*——那些在末日降临之夜没有醒来、却以某种方式活下来的人，被称为梦游者。

梦游者不是英雄，也不是救世主。他们只是……活下来了。

### 2.2 世界地图结构

- **五片派系大陆**：每片大陆由一个派系主导，拥有独特地形、生物、建筑配件；
- **中心混战大陆（灰心岛）**：中立地带，五大派系均在此留有势力，资源丰富但危险极高；
- **移动城堡**是玩家穿越各大陆的交通工具与战斗核心。

### 2.3 世界规则

- 派系之间没有法律，只有力量与利益；
- 梦游者是例外：因为"不属于任何人"，各派系都愿意与梦游者交易，但也都不真正信任他；
- 生物并非全是怪物——许多生物只是乱世中失去归属的存在，与梦游者的关系更像是"雇佣"和"共生"。

## 4. 设计支柱

本版新增。以下三条是不可妥协的核心玩家体验，所有设计决策必须通过这三条检验：

### 支柱 1：多样性是策略的根源

> 玩家不会因为"数值不够"而卡关，只会因为"搭配不对"而卡关。  
> 生物没有等级、没有进化、没有数值成长。每只生物的定位从获得那一刻就固定。  
> 游戏的深度来自"你有多少种生物"和"你选择哪几只上场"。

**检验标准**：如果某个机制让玩家觉得"我需要把这只生物练到更强"，那它违反了本支柱。

### 支柱 2：死亡让选择有重量

> 生物 HP 归零即死亡。复活代价高昂（金币+稀有道具）。  
> 每次出战前选择阵容时，玩家必须权衡：这只生物会不会死？我要带治疗手段吗？  
> 治疗是有的，但需要时机判断和资源投入——不是免费的安全网。

**检验标准**：如果某个机制让死亡变得"无所谓"（比如复活太便宜、死亡无惩罚），那它违反了本支柱。

### 支柱 3：化学反应创造惊喜

> 5 个派系的生物各有独特定位，跨派系组合会产生预料之外但可理解的化学反应。  
> 纯阵营队伍稳扎稳打，混搭队伍风险更高但爆发力更强。  
> 玩家通过试错和探索发现新组合，这个过程本身就是乐趣。

**检验标准**：如果某个派系组合的结果只是"数值+X%"而没有行为变化（比如新增 DOT、眩晕、召唤），那它不够惊喜。

## 5. 核心玩法循环

### 5.1 三层循环

```
【即时循环 (0-30秒)】
  观察战场 → 发现某只生物HP告急 → 决定是否使用治疗道具/技能 → 看到敌人被协同反应击杀
  紧张感来源：生物随时可能死亡，治疗窗口很短

【局循环 (3-10分钟)】
  选阵容（哪几只生物上阵） → 进战斗 → 3-5波敌人 → 通关/失败
  核心决策：阵容选择 = 风险博弈（强力生物vs保留安全？治疗位vs输出位？）
  结算：确定性生物奖励 + 金币 + 节点解锁

【长线循环 (小时-周)】
  收集更多生物种类 → 解锁新大陆节点 → 发现新化学反应 → 复活死亡生物（攒道具）→ 挑战更难关卡
  驱动力：生物图鉴完成度、化学反应发现记录、全大陆通关
```

### 5.2 完整游戏流程

```
主菜单 → 新游戏/继续
  → 世界地图（查看节点、管理生物健康状态、选择阵容）
  → 选择节点 → 进入战斗
    → 战前：部署生物到固定站位
    → 战中：观察+治疗+支援技能
    → 战后：结算奖励（确定性生物+金币）
  → 返回世界地图
    → 检查生物受伤状态
    → 治疗轻伤生物（简单/免费）
    → 复活死亡生物（金币+稀有道具）
    → 前往下一节点
  → 阵营商店（购买模块/道具）
  → 无限挑战（独立Roguelike循环）
```

## 6. 战斗系统

### 6.1 战斗模型

> **本版核心变更：从纯塔防变为生物有HP的站位塔防——敌人也会攻击你的生物，生物真的会受伤会死。**

| 战斗方   | 行为规则                     |
| ----- | -------------------------- |
| **你的生物** | 固定站位不动 → 自动攻击范围内的最近敌人     |
| **普通敌人** | 沿路线行进 → 到达生物附近时攻击最近的生物     |
| **特殊敌人** | 绕过生物 → 直攻城堡核心            |
| **城堡核心** | 被特殊敌人攻击 → HP归零则战斗失败       |

### 6.2 战斗流程

1. **战前部署**：查看敌方波次预览 → 选择上阵生物 → 将生物放置到战场上的固定站位格；
2. **战斗开始**：敌人沿路线行进，到达生物附近时敌人攻击生物、生物攻击敌人；
3. **即时决策**：
   - 观察各生物HP，判断是否需要使用治疗道具或治愈类技能；
   - 使用主角支援技能（减速、伤害、护盾等）；
   - 关注跨派系反应的触发时机；
4. **波次推进**：每波之间有 `[PLACEHOLDER]` 秒喘息，可查看下一波预览；
5. **结算**：胜利 → 获得确定性奖励 + 金币；失败 → 生物可能死亡，损失严重。

### 6.3 站位系统

- 战场上有若干**固定站位格**，玩家在战前将生物放入格子；
- 不同站位格有位置特性：

| 位置类型  | 特性                  | 策略意义       |
| ----- | ------------------- | ---------- |
| **前排格** | 近战范围，承受更多伤害         | 放高HP/治疗型生物 |
| **后排格** | 远程范围，较少被攻击          | 放脆皮输出型生物   |
| **核心格** | 保护城堡核心，特殊敌人必须经过此格才能攻击核心 | 放反精英型生物   |
| **侧翼格** | 攻击角度扩展，可覆盖多条路线      | 放AOE型生物    |

### 6.4 战斗中的生物状态

- 生物HP实时显示（血条），受伤阶段实时变化（健康→轻伤→重伤→濒死）；
- 生物HP归零时立即死亡——该站位格空出，无法在战斗中重新部署；
- 战斗结束后，存活生物保留受伤状态（带入下一局）；死亡生物进入"死亡状态"。

## 7. 生物系统

### 7.1 设计哲学：多样性 > 数值

> **本版核心变更：取消升级、取消进化/觉醒、取消碎片合成。生物的属性从获得那一刻起永远不变。**

每只生物的价值不在于"它有多强"，而在于"它能做什么别人做不到的事"。

### 7.2 生物属性

每只生物的属性是**固定常量**，不会因任何原因改变：

| 属性       | 说明                        |
| -------- | ------------------------- |
| **最大HP** | 固定值，决定这只生物能承受多少伤害         |
| **攻击力**  | 固定值，每次攻击造成的伤害             |
| **攻击速度** | 固定值，攻击间隔时间（秒）             |
| **攻击范围** | 固定值，近战/中程/远程/全图           |
| **目标优先级** | 固定策略：最近/血最低/精英优先/城堡优先     |
| **派系标签** | TECH / FAITH / NATURE / COMMERCE / MEMORY |
| **特殊能力** | 1个固定的被动或触发型能力             |
| **站位适配** | 前排/后排/侧翼/核心——决定该生物适合放哪个位置 |

### 7.3 生物定位分类

每只生物有一个清晰的定位标签，帮助玩家快速理解"这只生物干嘛用"：

| 定位     | 特征                | 代表（占位）      |
| ------ | ----------------- | ----------- |
| **输出型** | 高攻击力，低HP          | 机械狙击手（TECH） |
| **坦克型** | 高HP，低攻击力          | 铁壁机甲（TECH）  |
| **治疗型** | 能为友方回血            | 灵魂萤火（FAITH） |
| **控制型** | 减速/眩晕/定身敌人        | 荆棘兽（NATURE） |
| **爆发型** | 低频高伤，一击重创         | 回响行者（MEMORY）|
| **辅助型** | 增益友方/削弱敌方         | 废料赌徒（COMMERCE）|
| **召唤型** | 战斗中召唤临时单位         | （跨派系反应产物）   |
| **反精英型** | 对特殊敌人有额外伤害/拦截能力   | 圣装机兵（TECH×FAITH）|

> **设计目标**：每种定位至少有 2-3 只不同派系的生物可选，确保玩家在同一个战术需求上有多种选择。

## 8. 生物健康与生死系统

> **本版新增核心系统。**

### 8.1 受伤阶段

生物的HP不是简单的"满血/空血"，而是分为4个阶段，每个阶段有视觉和功能差异：

| 阶段  | HP范围                | 视觉表现      | 功能影响             |
| --- | ------------------- | --------- | ---------------- |
| 健康  | HP ≥ 75%            | 正常颜色      | 无                 |
| 轻伤  | 50% ≤ HP < 75%      | 颜色略暗淡     | 无数值惩罚，视觉提示"需要注意"  |
| 重伤  | 25% ≤ HP < 50%      | 明显闪烁/裂纹   | 无数值惩罚，但离死亡只差一步     |
| 濒死  | HP < 25%            | 红色警告光环    | 无数值惩罚，但随时可能死亡      |
| 死亡  | HP = 0              | 消失+死亡特效   | 该站位空出，生物进入死亡状态     |

> **关键设计决策**：受伤阶段**没有数值惩罚**（攻击力不降、速度不减）。惩罚是"离死亡更近"这件事本身。理由：符合"多样性>数值"支柱——如果受伤降数值，玩家会觉得生物"变弱了"，违反了"属性不变"原则。

### 8.2 跨局受伤继承

- 战斗结束后，存活生物的当前HP**保留**，带入下一局战斗；
- 轻伤生物可以通过**简单治疗**恢复到满HP（见第9章）；
- 重伤/濒死生物需要更多治疗资源；
- 这意味着玩家在连续闯关时必须管理生物健康状态——不能无限硬推。

### 8.3 死亡

- 生物在战斗中HP归零 = **立即死亡**，该站位空出，本场战斗无法补充；
- 死亡不是"暂时倒地"——生物真的死了，从阵容中消失；
- 死亡带来**情感冲击**和**策略后果**：失去一只独特定位的生物，可能让后续关卡缺少关键配合。

### 8.4 复活仪式

死亡生物可以通过**复活仪式**重新获得，但代价高昂：

| 复活要素     | 说明                          |
| -------- | --------------------------- |
| **金币**   | `[PLACEHOLDER]` 数量——基础经济代价  |
| **稀有道具** | 灵魂碎片/复活晶石——需从高难度关卡或混战大陆获取 |
| **无属性惩罚** | 复活后生物属性不变，HP恢复到满——符合"数值不变"原则 |

> 复活的昂贵代价确保死亡有重量感，但不让玩家陷入"永远无法挽回"的绝望。攒够了资源就能复活，这个过程本身就是一种长线目标。

## 9. 治疗系统

### 9.1 战斗中治疗

战斗中治疗是**防止生物死亡**的核心手段：

| 治疗来源     | 说明                          |
| -------- | --------------------------- |
| **治疗型生物** | 自动为范围内受伤友方回血（如灵魂萤火、圣花园产物）   |
| **治愈流技能** | 主角支援技能——主动释放，恢复指定生物HP      |
| **治疗道具** | 战前携带，战中手动使用——消耗型，数量有限      |

治疗型生物是**被动持续回血**，治愈流技能是**主动瞬间回血**，道具是**应急救命**。三种方式互补。

### 9.2 战斗外治疗（受伤恢复）

战斗结束后，受伤生物需要恢复才能以满HP进入下一局：

| 受伤阶段 | 恢复方式        | 成本       |
| ---- | ----------- | -------- |
| 轻伤   | 自动恢复（1局间隔）  | 免费       |
| 重伤   | 使用治疗道具      | `[PLACEHOLDER]` 金币 |
| 濒死   | 使用高级治疗道具    | `[PLACEHOLDER]` 金币 + 普通道具 |

## 10. 生物收集系统

> **本版核心变更：从随机掉落+好感度解锁 → 确定性奖励+副本刷取。**

### 10.1 设计原则

- **确定性**：玩家知道打哪个关卡能获得哪只生物，不会出现"打了100次还没掉"的挫败感；
- **可刷取**：每个关卡可以重复挑战，玩家可以回头刷缺少的生物；
- **收集驱动**：获得新生物 = 获得新战术选项，不是获得"更强版本"。

### 10.2 关卡奖励表

每个关卡有固定的生物奖励（首次通关获得，重复通关可获得同名生物的第二只）：

> 示例（占位）：

| 大陆  | 关卡节点       | 首次通关奖励      | 重复通关奖励      |
| --- | ---------- | ----------- | ----------- |
| 技术  | tech_battle_1 | 机械狙击手       | 金币 + 机械狙击手（可重复获得） |
| 技术  | tech_battle_2 | 铁壁机甲        | 金币 + 铁壁机甲   |
| 技术  | tech_boss   | 重型机甲（稀有）    | 金币 + 稀有道具   |
| 信仰  | faith_battle_1 | 灵魂萤火        | 金币 + 灵魂萤火   |
| ... | ...        | ...         | ...         |

### 10.3 同名生物的第二只

- 首次通关获得一只生物；
- 重复通关可以获得**同名生物的第二只**——这让玩家可以在一场战斗中部署两只同定位生物（比如双治疗、双坦克）；
- 这不是"碎片觉醒"，而是纯粹的"多一只同款"——两只机械狙击手的属性完全相同，但你可以同时放两只来覆盖两个方向；
- **设计意图**：同名生物的多只持有增加了部署灵活性，但不违反"数值不变"原则。

### 10.4 图鉴系统

- 收集进度可视化——每只生物有图鉴条目，显示：属性、定位、派系、特殊能力、所属大陆；
- 图鉴完成度是长线目标——"集齐5阵营全部生物"；
- 跨派系反应在图鉴中有专门的"反应发现"页面，记录玩家已触发的化学反应。

## 11. 城堡系统

### 11.1 城堡概念

移动城堡是玩家的**战场核心**和**交通枢纽**。

- 城堡有自己的HP（城堡核心HP）——被特殊敌人攻击时会受损；
- 城堡核心HP归零 = 战斗失败；
- 城堡有护盾（可恢复）和装甲（减少伤害）。

### 11.2 城堡模块系统

| 模块类型       | 功能              | 来源     |
| ---------- | --------------- | ------ |
| **站位格扩展模块** | 增加战场上的可部署站位格数量  | 主线推进   |
| **防御模块**   | 增加城堡核心HP和护盾     | 各大陆配件  |
| **能量模块**   | 提升主角技能能量恢复速度    | 各大陆配件  |
| **派系共鸣模块** | 装备后增强特定派系生物效果   | 特定派系大陆 |
| **侦察模块**   | 战前可查看更多敌方波次情报   | 混战大陆   |
| **仓库模块**   | 增加治疗道具携带上限      | 商人购买   |

### 11.3 站位格

> **本版变更：从"城堡上的槽位"变为"战场上的固定站位格"。**

- 战场上有 `[PLACEHOLDER]` 个站位格（初始4个，通过模块扩展到6-8个）；
- 站位格有位置特性（前排/后排/侧翼/核心），见6.3节；
- 生物放在站位格后固定不动，自动攻击范围内的最近敌人。
